# Memo Rewrite Lite

한국어 문장을 짧고 담백한 메모체로 바꿀 때 우선 적용하는 로컬 규칙.
지금은 아래 규칙을 기본으로 쓰고, 이후 필요하면 항목을 계속 추가하거나 수정하면 됨.

## 핵심 목표
- 의미는 유지하고, 어조는 한 단계 중립적으로 낮춤
- 설명문보다는 관찰 메모처럼 짧게 정리
- 기술 문맥과 용어는 보존

## 우선 규칙

### 1. 중립 어조 유지
- 단정형 서술은 완곡한 메모체 표현으로 낮춤.
- 확정적 판단보다는 관찰, 추정, 인상에 가까운 표현을 우선 사용.
- 우선 종결:
  - `~보임`
  - `~같음`
  - `~것 같음`
  - `~가까움`
  - `~해보임`
- 예:
  - `~위험하다` -> `~위험해보임`
  - `~편이 맞다` -> `~보임`
  - `~명확합니다` -> `~명확해보임`
  - `~역할을 합니다` -> `~역할을 함` 또는 `~역할을 할 수 있어 보임`

### 2. 단문 메모체로 변환
- 문장을 짧게 끊고, 연결 표현과 수식어는 줄임.
- 장황한 연결 문구는 삭제하거나 제목형 표현으로 압축.
- 우선 종결:
  - `~함`
  - `~됨`
  - `~봄`
  - `~보임`
  - `~해봄`
  - `~적합해보임`
  - `~가까움`
  - `~쓸만함`
- 예:
  - `핵심 패턴은 일관적입니다.` -> `핵심 패턴은 일관적으로 보임.`
  - `한 줄 결론으로 줄이면 이렇습니다.` -> `한 줄 결론,`
  - `실무 포지셔닝도 5개 결과를 종합하면 명확합니다.` -> `실무 포지셔닝은 5개 결과를 종합했을때,`

### 3. 구조 유지
- 원문에 있는 문단 구조, 불릿 구조, 비교 구조는 유지.
- 문단 제목이나 도입 문장은 더 짧게 줄여도 되지만, 뒤에 오는 불릿의 순서와 대응 관계는 유지.
- 이미 충분히 짧은 라벨형 불릿은 과하게 손대지 않음.
- 예:
  - `Figma MCP: 실제 구현 기준, truth source`
  - `OpenPencil: 구조 추출, skeleton 파악, 전처리용 parser`

### 4. 기술 용어와 고유명사 보존
- 제품명, 컴포넌트명, 영어 기술 용어는 번역하거나 풀어쓰지 않음.
- `OpenPencil`, `Figma MCP`, `Alert Overlay`, `AppLayout`, `truth source`, `parser`, `semantic content` 같은 표현은 그대로 유지.
- 의미가 흐려질 정도로 일반어로 바꾸지 않음.

### 5. 의미 축약 방식
- 설명형 문장을 관찰 메모형으로 압축하되, 정보는 버리지 않음.
- 다음 방향을 우선 사용:
  - `~잘 읽습니다` -> `~잘 읽음`
  - `~의미가 흔들립니다` -> `~혼란스러워짐`
  - `~빠집니다` -> `~누락됨`
  - `~맞습니다` -> `~쪽에 가까움`
  - `~강합니다` -> `~쓸만함` 또는 `~강해보임`

### 6. 피해야 할 변환
- 문장이 깨지거나 일부 텍스트가 유실되면 안 됨.
- 조사와 띄어쓰기를 어색하게 바꾸지 않음.
- 기술 용어를 임의로 순화하거나 다른 의미로 바꾸지 않음.
- 불릿을 문장으로 합치거나, 반대로 문장을 과도하게 쪼개지 않음.
- 지나친 통신체, 과도한 생략, 과장된 구어체는 피함.

## 예시 패턴

### 관찰형 전환
- `핵심 패턴은 일관적입니다.` -> `핵심 패턴은 일관적으로 보임.`
- `실무 포지셔닝도 ... 명확합니다.` -> `실무 포지셔닝은 ...`

### 단정형 완화
- `“구조는 읽는데 의미는 못 살린다”가 맞습니다.` -> `“구조는 읽는데 의미는 못 살림” 쪽에 가까움.`
- `안정적인 truth source 역할을 합니다.` -> `안정적인 truth source 역할을 할 수 있어 보임.`

### 압축형 도입
- `한 줄 결론으로 줄이면 이렇습니다.` -> `한 줄 결론,`

## 출력 규칙
- 핵심 의미 유지
- 한국어만 출력
- 부가 설명 금지
- 문단 수, 불릿 수, 순서는 가능하면 유지
- 과도한 통신체, 구어체, 불릿화는 피함
